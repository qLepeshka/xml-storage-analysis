# Инструмент сравнительного анализа методов хранения XML — Сводка по проекту

## Обзор проекта
Разработано комплексное программное приложение для сравнительного анализа различных методов хранения XML-данных в реляционных базах данных. Приложение оценивает нативное текстовое хранение, нормализованное реляционное хранение, хранение с использованием XML-типов данных и гибридные подходы.

## Ключевые реализованные возможности

### 1. Поддержка нескольких баз данных
- Интеграция с PostgreSQL
- Интеграция с MySQL
- Интеграция с SQL Server
- Унифицированный слой подключения к базам данных

### 2. Методы хранения
- **Нативное текстовое хранение**: Хранение XML в виде простого текста
- **Нормализованное реляционное хранение**: Разбор XML в реляционные таблицы
- **Хранение с XML-типом данных**: Использование специализированных XML-типов данных
- **Гибридный подход**: Комбинирование нескольких методов для оптимальной производительности

### 3. Тестирование производительности
- Автоматическое тестирование операций чтения, записи, обновления и поиска
- Статистический анализ метрик производительности
- Настраиваемые параметры тестирования
- Множественные итерации тестирования для повышения точности

### 4. Визуализация
- Интерактивные диаграммы и графики
- Визуализации сравнения производительности
- Тепловые карты для кросс-методного анализа
- Анализ трендов

### 5. Отчётность
- Комплексные HTML-отчёты
- Рекомендации по производительности
- Руководства по выбору метода
- Подробная статистика

### 6. Управление данными
- Импорт/экспорт тестовых наборов данных
- Поддержка форматов XML, JSON, CSV и Excel
- Возможности валидации данных

### 7. Управление конфигурацией
- Файлы конфигурации JSON/YAML
- Гибкие настройки параметров
- Конфигурация подключения к базе данных
- Настройка параметров тестирования

## Архитектура

Приложение следует модульной архитектуре с чётким разделением ответственности:

- `app/application.py` — Главный оркестратор приложения
- `app/database_manager.py` — Слой подключения к базе данных
- `app/storage_methods.py` — Реализации методов хранения
- `app/performance_tester.py` — Фреймворк тестирования производительности
- `app/visualizer.py` — Компоненты визуализации
- `app/report_generator.py` — Генерация отчётов
- `app/data_handler.py` — Функциональность импорта/экспорта данных
- `app/config_manager.py` — Управление конфигурацией

## Использование

Приложение может работать в двух режимах:
1. Интерактивный режим с меню-интерфейсом
2. Режим командной строки с конкретными действиями

Пример использования:
```bash
python main.py config.json --action test
```

## Зависимости

Приложение требует следующие пакеты Python:
- Коннекторы баз данных (psycopg2, PyMySQL, pyodbc)
- SQLAlchemy для ORM
- Обработка XML (lxml)
- Анализ данных (pandas, NumPy)
- Визуализация (matplotlib, seaborn, plotly)
- Веб-фреймворк (Flask)
- Поддержка форматов файлов (openpyxl)

## Заключение

Инструмент сравнительного анализа методов хранения XML предоставляет комплексное решение для оценки различных стратегий хранения XML в реляционных базах данных. Он предлагает возможности автоматизированного тестирования, визуализации и отчётности, чтобы помочь пользователям принимать обоснованные решения о наиболее подходящем методе хранения для их конкретных случаев использования.

Модульная архитектура позволяет легко расширять приложение дополнительными методами хранения, системами баз данных или техниками визуализации. Система конфигурации обеспечивает гибкость для различных сценариев и сред тестирования.
