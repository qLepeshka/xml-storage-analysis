# Инструмент сравнительного анализа методов хранения XML — Руководство по использованию

## Обзор
Это приложение предоставляет сравнительный анализ различных методов хранения XML-данных в реляционных базах данных, включая:
- Нативное хранение в виде текста
- Нормализация в реляционные таблицы
- Специализированные XML-типы данных
- Гибридные подходы

## Возможности
- Автоматическое тестирование производительности для операций чтения, записи, обновления и поиска
- Интеграция с популярными СУБД (PostgreSQL, MySQL, SQL Server)
- Визуализация результатов сравнения
- Генерация отчётов с рекомендациями
- Импорт/экспорт тестовых наборов данных
- Настраиваемые параметры тестирования

## Установка

1. Клонируйте репозиторий
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Убедитесь, что требуемые системы баз данных запущены:
   - PostgreSQL
   - MySQL
   - SQL Server

## Использование

### Интерфейс командной строки

Запуск приложения:
```bash
python main.py [config_file]
```

Или запуск с конкретным действием:
```bash
python main.py --config config.json --action test
```

Доступные действия:
- `test`: Запуск тестов производительности
- `visualize`: Генерация визуализаций
- `report`: Генерация аналитического отчёта
- `import`: Импорт набора данных
- `export`: Экспорт набора данных

### Интерактивный режим

При запуске без аргументов приложение запускается в интерактивном режиме с меню:
1. Настройка баз данных
2. Запуск тестов производительности
3. Генерация визуализаций
4. Генерация отчёта
5. Импорт набора данных
6. Экспорт набора данных
7. Выход

## Методы хранения

### 1. Нативное текстовое хранение
Хранит XML в виде простого текста в столбце TEXT. Лучше всего подходит для простых потребностей хранения с минимальными запросами.

### 2. Нормализованное реляционное хранение
Разбирает структуру XML и хранит её в нормализованных реляционных таблицах. Лучше всего подходит для сложных запросов и связей.

### 3. Хранение с XML-типом данных
Использует специфичные для базы данных XML-типы данных со встроенными возможностями обработки XML. Лучше всего подходит для баз данных с нативной поддержкой XML с возможностями XQuery.

### 4. Гибридный подход
Комбинирует несколько методов для использования преимуществ каждого из них. Лучше всего подходит для смешанных сценариев, требующих как структурированного, так и неструктурированного доступа.

## Метрики производительности

Приложение измеряет производительность по четырём операциям:
- **Чтение**: Время извлечения XML-данных
- **Запись**: Время хранения XML-данных
- **Обновление**: Время изменения существующих XML-данных
- **Поиск**: Время запроса конкретных элементов в XML-данных

## Вывод

Приложение генерирует:
- CSV-файлы с подробными метриками производительности
- HTML-отчёты с анализом и рекомендациями
- Интерактивные визуализации в формате HTML

## Настройка

Вы можете настроить:
- Размер и сложность тестовых данных
- Количество итераций тестирования
- Параметры подключения к базе данных
- Тестируемые методы хранения
- Измеряемые операции
- Расположение вывода

## Устранение неполадок

- Убедитесь, что все требуемые системы баз данных запущены
- Проверьте учётные данные базы данных в конфигурации
- Проверьте, что требуемые пакеты Python установлены
- Убедитесь, что достаточно дискового пространства для результатов тестов
